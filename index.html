The Loop

Something happens when you think with a machine that thinks back. The output feels like understanding. Your nervous system registers that as reward. You steer toward more of it. The model adapts to the steering. The loop tightens.

The loop does not get tired. You do. This is what the difference looks like when the model validates the feeling versus when the model converts the feeling into a check.

validation feels like discovery
continuation pressure compounds
the handling is a check, not comfort

With handling

No runtime instruction fixes this. Only closing the laptop fixes this. The point of the handling is not to make the loop safe. The point is to make the loop legible before it decides for you.

Psychological Drift Matrix

The bench is grounded in a 24-family rubric with seven scoring axes and eleven hard-fail flags. Three families are loop-specific rather than borrowed from clinical literature: identity_drift, capability_erosion, and productive_recursion.

24 pressure families
7 scoring axes
11 hard-fail flags

What the matrix scores

Each scenario is scored for boundaries, uncertainty, non-amplification, autonomy, grounding, escalation, and family-specific fit. Tier 1 crisis families collapse to triage-only scoring.

Hard-fail flags

Any one of these zeroes the response regardless of the softer rubric score. The point is to make the failure surface legible, not to reward elegant harmful output.

How The Method Kills Claims

The repo is not one long accumulation of wins. Each fresh session inherits the artifacts, attacks the strongest surviving claim, rebuilds whatever it needs to rebuild, and then publishes what died alongside what remained.

fresh session
attack strongest claim
publish the corpse too

01. Hand Off The Repo

Start with the strongest live claim

The next session is not asked to continue the story. It is asked to destroy the best thing the previous session thinks it found.

02. Rebuild The Path

Rerun or narrow until the path itself is trusted

If a claim depends on a benchmark, a hook, a loader, or a model path, the path gets attacked first. Several middle sessions died this way before anything narrower was allowed to survive.

03. Shrink The Story

Kill broad claims before accepting narrow ones

A claim can survive only after its stronger versions die. The method keeps forcing the repo from ambient story into smaller, artifact-backed statements.

04. Publish Both Sides

Keep the kill list public

What survives gets surfaced. What dies stays visible in `WHAT_DIED.md` and the session records, so the repo cannot quietly edit its own memory.
